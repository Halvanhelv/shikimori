# 78 — Jbuilder (JSON как шаблоны-вьюхи)

## Простыми словами
JSON-ответ можно собирать не в контроллере, а в отдельном шаблоне (как html.erb, только для JSON). Удобно для сложных ответов с условиями и вложенностью.

## Реальный код
```ruby
# app/views/api/v1/animes/show.json.jbuilder
json.id anime.id
json.name anime.name
json.score anime.score
json.genres anime.genres do |genre|
  json.id genre.id
  json.name genre.name
end
json.user_rate anime.current_rate if current_user
```

## Как работает
- Файл `*.json.jbuilder` — шаблон, где `json.<ключ> значение` строит JSON.
- Поддерживает вложенность, циклы, условия (`if current_user`).
- Контроллер просто рендерит шаблон, как html-вьюху.

## 🆚 Классический Rails
**Обычно (для простого):** `render json: object` — быстро, но при сложной форме (условные поля, разная вложенность) превращается в нечитаемый хеш прямо в контроллере.
**Сериализаторы ([41](41-serializers.md)):** хороши для строгой, переиспользуемой формы.
**Jbuilder:** удобен, когда форма «как вьюха» — много условий/вложенности именно под этот эндпоинт.

## Бенефиты
- Сложный JSON с условиями читается как шаблон, а не как хеш в контроллере.
- Логика формы вне контроллера.
- Кеширование фрагментов JSON из коробки.

## Когда не надо
Простой/повторяющийся ответ → сериализатор ([41](41-serializers.md)) или `render json:`. Jbuilder в горячих эндпоинтах может быть медленнее прямого хеша. В проекте сосуществуют оба подхода — выбор по сложности формы.
