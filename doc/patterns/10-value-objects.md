# 10 — Value Objects

## Простыми словами
Value Object — это объект-значение, который важен своим содержимым, а не тем, «кто он»: две одинаковые суммы денег равны, как две одинаковые купюры. У него нет id и его не меняют — нужно другое значение, создаёшь новое. Как 100 рублей: неважно, какая именно банкнота, важна сумма.

## Что
Объект-значение: сравнивается по содержимому, иммутабелен, без идентичности (нет `id`). Инкапсулирует логику про «значение».

## Реальный код

## Признаки value-объекта
1. Сравнение по значению (не по id/ссылке).
2. Иммутабельность — нужно другое, создаёшь новый.
3. Без идентичности — не сущность БД.
4. Самодостаточен — вся логика про значение внутри.

## VO vs Entity
| | Value Object | Entity |
|--|--|--|
| Идентичность | нет | есть (`id`) |
| Сравнение | по значению | по id |
| Меняется | нет (новый) | да (update) |
| Пример | `SpentTime`, `IncompleteDate` | `User`, `Anime` |

## Зачем / где полезно
Когда значение имеет правила и трогается в 2+ местах: считаешь, форматируешь, сравниваешь. Логика собирается в одном месте, нет дублей магических чисел, тест без БД. Современный способ — `Data.define`. См. также [06 — ShallowAttributes](06-shallow-attributes.md).

## 🆚 Классический Rails
**Обычно:** значения вроде «потраченного времени» таскают как сырое число и форматируют/считают прямо в хелперах и вьюхах. Логика и магические числа (365, 30) дублируются по проекту.
**Здесь:** значение завёрнуто в отдельный класс (`SpentTime`) со сравнением по содержимому и всей логикой внутри — `SpentTime.new(days: 400).years`.

## Бенефиты
- Логика про значение собрана в одном месте, без дублей.
- Сравнение по содержимому — удобно для тестов и коллекций.
- Тестируется без базы данных.
- Иммутабельность исключает случайные мутации общего значения.
