# 39 — Кеширование на границе декоратора

## Простыми словами
Карточка аниме показывает кучу блоков: роли, похожее, оценки друзей — и каждый требует своего запроса в базу. Если вьюха дёрнет такой метод 3 раза, без кеша будет 3 запроса. Декоратор запоминает результат первого вызова и отдаёт его дальше бесплатно — как официант, который один раз сходил на кухню и держит блюдо под рукой, а не бегает за ним каждый раз.

## Что
Декораторы (Draper) — слой презентации, где агрессивно кешируются дорогие запросы, чтобы вьюха дёргала их много раз бесплатно.

## Реальный код
```ruby
class AniMangaDecorator < DbEntryDecorator
  prepend ActiveCacher.instance
  instance_cache :critiques_count, :reviews_count, :current_rate, :chronology,
    :roles, :related, :friend_rates, :external_links, :topic_views

  def chronology = Animes::ChronologyQuery.new(object).fetch   # дорогой запрос
  def roles      = RolesQuery.new(object)
end
```

## Как работает
- Декоратор оборачивает модель (`object`) и добавляет презентационную логику + хелперы (`Draper::ViewHelpers`).
- Дорогие методы кешируются через [instance_cache](03-instance-cache-prepend.md): первый вызов считает, дальше — из ivar.
- Запросы инкапсулированы в query-объекты ([07](07-query-object-base.md)), декоратор их склеивает.
- Граница: модель — данные, query — запросы, декоратор — представление + кеш.

## Зачем / где полезно
Страницы с десятками вычисляемых блоков (карточка аниме: роли, похожее, хронология, оценки друзей). Кеш на границе вьюхи → один рендер не бьёт по БД повторно за тем же. Альтернатива в Rails 8 — ViewComponent + явная мемоизация.

## 🆚 Классический Rails
**Обычно:** вычисляемое значение мемоизируют прямо в модели или хелпере вручную (`@roles ||= ...`), а презентационные методы складывают в `app/helpers` без привязки к объекту.
**Здесь:** презентация живёт в декораторе (Draper), а кеш дорогих методов объявляется декларативно одной строкой `instance_cache :roles, :chronology, ...` — не нужно вручную писать `||=` в каждом методе. Граница чёткая: модель — данные, query — запросы, декоратор — представление + кеш.

## Бенефиты
- Один запрос на блок за рендер вместо повторных при многократном вызове.
- Кеш объявляется списком, без ручного `||=` в каждом методе.
- Модель не засоряется презентационными методами.
- Дорогие запросы вынесены в query-объекты, декоратор их только склеивает.
