# 70 — NamedLogger (отдельные лог-файлы под подсистемы)

## Простыми словами
Вместо одной свалки в `log/development.log` важные подсистемы (отправка писем, faye, парсеры) пишут в свои отдельные файлы. Проще искать «что случилось с письмами».

## Реальный код
```ruby
# lib/named_logger.rb даёт именованные логгеры:
NamedLogger.email.info "failed to send email to #{user.email}"      # → log/email.log
NamedLogger.faye_publisher.info "#{data.to_json} #{channels}"        # → log/faye_publisher.log
```

## Как работает
- `NamedLogger.<name>` создаёт/возвращает логгер, пишущий в отдельный файл `log/<name>.log`.
- Каждая подсистема логирует в свой канал.
- Общий апп-лог не засоряется деталями интеграций.

## 🆚 Классический Rails
**Обычно:** всё валится в `Rails.logger` → один гигантский файл, где письма, SQL, запросы и фон перемешаны. Дебажить интеграцию = `grep` по шуму.
**Здесь:** у каждой подсистемы свой файл. Проблема с faye → смотришь `faye_publisher.log`, а не весь лог.

## Бенефиты
- Быстрый дебаг конкретной подсистемы.
- Можно настроить отдельную ротацию/уровень на канал.
- Меньше шума в основном логе.

## Когда не надо
Маленький проект — хватит `Rails.logger` с тегами (`tagged`). Отдельные файлы окупаются, когда подсистем много и они шумные.
