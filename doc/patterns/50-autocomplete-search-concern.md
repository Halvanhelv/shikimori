# 50 — Автокомплит: тонкий контроллер + Search-объект

## Простыми словами
Контроллер здесь — как официант: принял заказ (фразу из строки поиска), отнёс на кухню (Search-объект) и вынес тарелку (JSON). Готовкой он не занимается — вся логика поиска на «кухне», отдельно. А повторяющийся кусок «как принять заказ» вынесен в общий блокнот (concern), чтобы каждый официант не переписывал его заново.

## Что
Контроллеры автодополнения предельно тонкие: вся логика — в отдельных `Search::*`-объектах, общий ввод — через concern.

## Реальный код
```ruby
class Autocomplete::FansubbersController < ShikimoriController
  include SearchPhraseConcern        # даёт search_phrase из params

  def index
    render json: Search::Fansubber.call(
      phrase: search_phrase,
      ids_limit: Autocomplete::AutocompleteBase::LIMIT,
      kind: params[:kind]
    )
  end
end
```

## Как работает
- Контроллер делает ровно одно: достаёт фразу (`SearchPhraseConcern`), зовёт callable Search-объект ([01](01-method-object.md)), рендерит JSON.
- Логика поиска (Elasticsearch/нормализация/лимиты) — в `Search::Fansubber`, не в контроллере.
- Общий ввод вынесен в concern → не дублируется по всем автокомплит-контроллерам.

## Зачем / где полезно
Эталон тонкого контроллера: input (concern) → callable (Search-объект) → output (JSON). Логика тестируется без HTTP, контроллеры однотипны и крошечные. Подходит для любых «лёгких» JSON-эндпоинтов (автокомплит, виджеты, лукапы).

## 🆚 Классический Rails
**Обычно:** логику поиска (запросы к Elasticsearch/БД, нормализацию фразы, лимиты) пишут прямо в экшене контроллера. Контроллер пухнет, тестировать его можно только через HTTP, а похожий код копируется по всем автокомплит-экшенам.
**Здесь:** контроллер делает ровно три шага — достать фразу через concern, позвать callable `Search::Fansubber.call(...)`, отрендерить JSON. Вся «начинка» поиска живёт в отдельном Search-объекте, общий ввод — в concern.

## Бенефиты
- Логику поиска тестируешь как обычный объект, без поднятия HTTP-стека.
- Контроллеры однотипные и крошечные — меньше места для багов.
- Общий парсинг ввода не дублируется (DRY через concern).
- Search-объект переиспользуется вне контроллера (джоба, консоль, другой эндпоинт).
