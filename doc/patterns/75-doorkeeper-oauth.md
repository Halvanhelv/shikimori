# 75 — Doorkeeper (OAuth для своего API)

## Простыми словами
Чтобы сторонние приложения (мобильный клиент, бот) могли ходить в твой API от имени юзера — нужен OAuth. Doorkeeper добавляет это в Rails: выдаёт токены, проверяет их, ограничивает доступ скоупами.

## Реальный код
```ruby
# API-контроллер требует OAuth-токен со скоупом:
class Api::V1Controller < ApplicationController
  doorkeeper_for :all          # все экшены требуют валидный токен
  # ...
end

# В описании метода (apipie):
# description 'Requires `user_rates` oauth scope'
```

## Как работает
- Doorkeeper хранит OAuth-приложения и токены, реализует стандартный flow (authorize → token).
- Клиент шлёт `Authorization: Bearer <token>`; Doorkeeper проверяет токен и его **скоупы** (`user_rates`, `comments`...).
- Без валидного токена/скоупа → 401.

## 🆚 Классический Rails
**Обычно (наивно):** проверяют API-ключ в параметре или сессию-куку — небезопасно, нет ограничения прав, нельзя отозвать доступ конкретному приложению.
**Здесь:** стандартный OAuth2 — токены с ограниченными скоупами, отзыв, срок жизни, отдельные приложения. Юзер даёт боту только нужные права.

## Бенефиты
- Стандарт (любой OAuth-клиент работает).
- Скоупы → приложение получает только нужные права, не всё.
- Токены отзываемы, с TTL; не светим пароль/сессию.

## Когда не надо
Если API только для своего фронта (тот же домен) — хватит сессии/CSRF. OAuth нужен для **сторонних** клиентов и публичного API.
