# 83 — Пул прокси для скрапинга

## Простыми словами
Когда часто качаешь данные с чужого сайта, он начинает блокировать твой IP. Решение — крутить запросы через пул прокси-серверов, чтобы выглядеть как разные клиенты.

## Реальный код
```ruby
# базовый парсер (см. 46) ходит через Proxy:
def get(url, required_text = @required_text)
  Proxy.get(
    url,
    timeout: 30,
    required_text: required_text,     # проверка, что страница валидна (не капча)
    no_proxy: Rails.env.test? ? true : @no_proxy,
    log: @proxy_log
  )
end
```

## Как работает
- `Proxy.get` берёт прокси из пула, шлёт запрос через него.
- `required_text` — ожидаемый кусок текста на странице: нет его (бан/капча) → прокси считается плохим, берётся следующий.
- В тестах `no_proxy: true` — ходим напрямую, без реальных прокси.

## 🆚 Классический Rails
**Обычно (наивно):** `Net::HTTP.get(url)` напрямую с одного IP → сайт банит после N запросов, импорт встаёт.
**Здесь:** ротация прокси + проверка валидности ответа + таймауты → устойчивый скрапинг, обход блокировок и капчи.

## Бенефиты
- Обход IP-блокировок при массовом импорте.
- Авто-отбраковка «битых» ответов (капча/бан) по `required_text`.
- Таймауты не дают зависнуть на медленном прокси.

## Когда не надо
Если у источника есть **официальный API** — используй его, не скрапь. Прокси-пул — крайнее средство для сайтов без API. Соблюдай ToS/легальность источника.
