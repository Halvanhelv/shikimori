# 31 — `enumerize` (типизированные enum)

## Простыми словами
Позволяет хранить в поле не любую строку, а только одно из заранее разрешённых значений (например, статус аниме: «онгоинг», «вышло»), и сразу даёт удобные проверки вроде `anime.ongoing?`. Плюс автоматом подтягивает человекочитаемые названия на нужном языке. Это как выпадающий список вместо поля для свободного ввода — выбрать можно только из готовых вариантов.

## Что
Enum'ы с предикатами и i18n поверх атрибута, со значениями из `dry-types`.

## Реальный код
```ruby
class Anime < ApplicationRecord
  enumerize :status, in: Types::Anime::Status.values, predicates: true
  enumerize :rating, in: Types::Anime::Rating.values, predicates: true
  enumerize :kind,   in: Types::Anime::Kind.values,   predicates: true
end

# даёт предикаты:
anime.ongoing?   # true/false
anime.tv?
```

## Как работает
- `in:` — допустимый набор (берётся из `dry-types` enum, см. [05](05-dry-types-boundaries.md)).
- `predicates: true` — автогенерит `anime.ongoing?` и т.п.
- Интеграция с i18n — человекочитаемые названия значений.
- В отличие от Rails-enum (целочисленный маппинг), `enumerize` гибче с источниками значений и хелперами.

## Зачем / где полезно
Статусы/типы/рейтинги с предикатами и локализацией. Используется в 50+ моделях проекта. Один источник значений (dry-types) + удобные хелперы в коде и вьюхах.

## 🆚 Классический Rails
**Обычно:** используют встроенный `enum` Rails, который маппит значения на целые числа в БД и не умеет из коробки в i18n; либо хранят сырые строки без всякой валидации набора значений.
**Здесь:** `enumerize` берёт допустимый набор из `dry-types`, автогенерит предикаты (`anime.ongoing?`) и интегрируется с i18n — один источник значений на все модели.

## Бенефиты
- Предикаты вроде `anime.tv?` из коробки, без ручных методов.
- Локализованные человекочитаемые названия значений (i18n).
- Один источник истины для допустимых значений (dry-types), переиспользуемый в 50+ моделях.
- Гибче встроенного Rails-enum: значения не привязаны к целочисленному маппингу.
