# 82 — BanDuration: парсинг и формат значения

## Простыми словами
Длительность бана можно задать строкой «2w 3d» (2 недели 3 дня) или числом минут. Value-объект `BanDuration` понимает оба формата и умеет красиво показать обратно.

## Реальный код
```ruby
class BanDuration
  delegate :to_i, :zero?, :minutes, to: :value
  attr_reader :value

  def initialize(duration)
    @value = duration.is_a?(String) ? parse(duration) : duration   # строка ИЛИ число минут
  end

  def to_s          # 1500 минут → "1d 1h"
    hash = to_hash
    result = []
    result << "#{hash[:days]}d"  if hash[:days].positive?
    result << "#{hash[:hours]}h" if hash[:hours].positive?
    # ...
    result.any? ? result.join(' ') : '0m'
  end
end

BanDuration.new('2w 3d').to_i    # минуты
BanDuration.new(1500).to_s       # "1d 1h"
```

## Как работает
- Конструктор принимает разные входы (строка-формат / число) — нормализует в минуты.
- `to_s`/`humanize` собирают обратно человекочитаемую строку.
- `delegate ... to: :value` — объект ведёт себя как число там, где надо (`to_i`, `zero?`).

## 🆚 Классический Rails
**Обычно:** длительность таскают как голое число минут, а парсинг «2w 3d» и форматирование «1d 1h» дублируют в контроллере, вьюхе, админке.
**Здесь:** вся логика «длительности» в одном value-объекте ([10](10-value-objects.md)). Парсинг ввода и красивый вывод — в одном месте.

## Бенефиты
- Парсинг и форматирование не дублируются.
- Объект ведёт себя как число (delegate) — удобно в расчётах.
- Тестируется без БД: `BanDuration.new('2w').to_i`.

## Когда не надо
Если длительность — просто число и нигде не парсится/форматируется — value-объект избыточен. Окупается при наличии нетривиального ввода/вывода.
