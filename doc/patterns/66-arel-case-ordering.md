# 66 — Сортировка по своему порядку (Arel.sql + CASE)

## Простыми словами
Иногда нужно отсортировать не по алфавиту/числу, а по **своему** порядку категорий (сначала опенинги, потом эндинги, потом клипы). Делается через SQL `CASE` прямо в `order`.

## Реальный код
```ruby
class Video < ApplicationRecord
  scope :ordered, -> {
    order(
      Arel.sql(
        <<~SQL.squish
          case kind
            #{kind.values.map.with_index { |v, i| "when '#{v}' then #{i}" }.join("\n")}
            else 99999999999
          end,
          id
        SQL
      )
    )
  }
end
```

## Как работает
- `CASE kind WHEN 'op' THEN 0 WHEN 'ed' THEN 1 ... END` — присваивает каждой категории числовой вес.
- Сортировка по этому весу → нужный порядок категорий, внутри — по `id`.
- Список `kind.values` (из enum) генерит ветки `WHEN` автоматически — добавил тип, порядок подхватился.
- `Arel.sql(...)` — обязательная обёртка: Rails требует явно пометить «доверенный» сырой SQL в `order`.

## 🆚 Классический Rails
**Обычно:** `order(:kind)` отсортирует по алфавиту — не то. Делают маппинг в Ruby и сортируют массив в памяти (`sort_by`) → грузят всё, теряют пагинацию/индексы.
**Здесь:** порядок считается в БД одним запросом, работает с `LIMIT/OFFSET`, не тянет всё в память.

## Бенефиты
- Кастомный порядок прямо в SQL → совместимо с пагинацией и индексами.
- Веса генерятся из enum → один источник правды, не разъезжается.

## Когда не надо
Если порядок — просто поле (`position`), используй `acts_as_list` ([32](32-acts-as-list.md)). CASE — для порядка по категориям без отдельной колонки.

⚠️ Сырой SQL в `order` только через `Arel.sql` и только из доверенных данных (тут — значения enum, не пользовательский ввод).
