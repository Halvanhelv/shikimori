# 20 — `sidekiq-limit_fetch` (лимит конкуренции очередей)

## Что
Ограничивает, сколько задач из конкретной очереди выполняется одновременно. Защищает тяжёлые ресурсы.

## Реальный код
```yaml
# config/sidekiq.yml
:limits:
  cpu_intensive: 2        # максимум 2 параллельно
  torrents_parsers: 1     # строго по одному
  webm_thumbnails: 1
  mal_parsers: 40
  achievements: 50
  imports: 2
  dangerous_actions: 1
```

## Как работает
- Глобальный лимит на очередь поверх обычной конкуренции Sidekiq.
- `torrents_parsers: 1` → парсер торрентов всегда один в моменте (внешний источник не любит параллель).
- `cpu_intensive: 2` → тяжёлые расчёты не съедят все треды.

## Зачем / где полезно
- Внешние API с лимитом коннектов.
- CPU/IO-тяжёлые задачи, которые нельзя пускать пачкой.
- Приоритизация: лёгкие очереди с большим лимитом, тяжёлые с малым.

Принцип: разделяй работу по очередям и дай каждой свой потолок пропускной способности.
