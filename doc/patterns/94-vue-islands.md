# 94 — Островки Vue (точечная интерактивность)

## Простыми словами
Большая часть страницы — обычный серверный HTML, но в нескольких местах нужна богатая интерактивность (форма с перетаскиванием, теги). Туда «вживляют» маленькое Vue-приложение — остров. Остальная страница про Vue ничего не знает.

## Реальный код
```slim
/ серверный HTML с «якорем» для Vue
#vue_external_links data-links=links.to_json
```
```js
// Vue монтируется только в этот узел
new Vue({ el: '#vue_external_links', /* ... */ })
```

## Как работает
- Сервер рендерит обычный HTML + пустой контейнер с данными в `data-*`.
- JS находит контейнер и монтирует туда Vue-компонент (форма, draggable-список, теги).
- Состояние внутри острова (Vuex), остальная страница — серверная ([93](93-turbolinks-ssr.md)).

## 🆚 SPA целиком
**SPA:** вся страница — Vue/React, сервер отдаёт пустой `<div id="app">` + JSON. Много JS, дубль логики, сложный роутинг/SEO.
**Острова:** Vue только там, где реально нужно (формы редактирования, интерактивные виджеты). 95% страницы — лёгкий серверный HTML, JS-бандл маленький.

## Бенефиты
- Богатая интерактивность только где нужно, без переписывания всего на SPA.
- Маленький JS, быстрый рендер, SEO остального контента сохранён.
- Серверная и клиентская части слабо связаны (через `data-*`).

## Когда не надо
Если интерактивна **вся** страница (сложное приложение) — острова станут лоскутным одеялом, проще полноценный SPA/Hotwire. Острова — для «в основном контент + пятна интерактива». Современный аналог идеи — Hotwire/Stimulus, Astro islands.
