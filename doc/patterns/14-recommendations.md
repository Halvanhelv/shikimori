# 14 — Рекомендации (коллаборативная фильтрация)

## Что
Подбор похожих аниме по оценкам юзеров. Алгоритмы (Pearson, Euclidean, SVD) написаны с нуля и подключаемы, с цепочкой нормализаций.

## Реальный код
```ruby
class RecommendationsWorker
  def perform(user_id, type, metric, threshold, cache_key, user_list_cache_key)
    Rails.cache.fetch(cache_key, expires_in: 2.weeks) do
      sampler = send(metric.to_sym, type.constantize, user_list_cache_key)  # выбор алгоритма
      sampler.recommend(user_id, threshold)
    end
  end
end

# Pearson — классическая формула: ковариация / (σ1 * σ2)
# SVD — кешированное разложение матрицы, лукап предсказанных оценок
```

## Как работает
- Метрика выбирается по имени (`metric` → метод) → легко A/B-тестировать.
- Каждый sampler оборачивает нормализацию (z-score центрирует вокруг среднего).
- Сравнивает юзеров, только если ≥ `MINIMUM_SHARED` (20) общих оценок.
- Результат кешируется в `Rails.cache` на 2 недели (расчёт дорогой → в фоне).

## Зачем / где полезно
Рекомендательные системы без внешних сервисов. Учит математике коллаборативной фильтрации (корреляция, SVD) и тому, как делать алгоритмы сменными через единый интерфейс. Дорогой расчёт → фон + кеш по ключу.
