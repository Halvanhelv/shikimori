# 13 — Турниры (Strategy + конечный автомат)

## Простыми словами
Турнир по голосованию за аниме можно проводить по-разному (на вылет, плей-офф, швейцарка), а сами матчи проходят стадии: создан, идёт, завершён. Алгоритм построения сетки выбирается как сменный модуль, а ход матчей рулит автомат состояний. Как спортивный турнир: правила сетки можно выбрать, но любой матч всё равно идёт «начался → играют → закончился».

## Что
Голосование за аниме навылет. Сетка строится по одной из трёх стратегий (double elimination / play-off / swiss), матчи проходят по состояниям, победитель проходит дальше.

## Реальный код
```ruby
class Contest::DoubleEliminationStrategy
  def total_rounds
    @total_rounds ||= Math.log(@contest.members.count, 2).ceil * 2   # двойное выбывание
  end

  def create_rounds
    total_rounds.times { |i| create_round(...) }
    @contest.rounds.each { |r| fill_round_with_matches(r) }
  end
end

# ежедневный воркер гоняет состояния:
class Contest::Progress   # created → started → freezed → finished
  # фризит матчи после finish_date, стартует pending, финишит проголосованные
end
```

## Как работает
- **Strategy**: `Contest` делегирует генерацию сетки стратегии; каждая считает раунды/пары по-своему.
- **Автоматы** на трёх уровнях: `Contest`, `ContestRound`, `ContestMatch` (aasm).
- Голоса — через `acts_as_votable` с кешем (`cached_votes_up`/`down`).
- Оркестрация — ежедневная задача через clockwork.

## Зачем / где полезно
Любой многоэтапный процесс с вариантами алгоритма: турниры, воронки, пайплайны обработки. Учит сочетанию Strategy + State Machine + cron-оркестрации. См. [22 — aasm](22-aasm-state-machines.md), [33 — acts_as_votable](33-acts-as-votable.md).

## 🆚 Классический Rails
**Обычно:** варианты алгоритма пишут как ветвление `if contest.type == :swiss ... elsif ...` внутри модели, а смену стадий — флагами вроде `started: true`. Логика разрастается в `case`/`if` и трудно расширяется.
**Здесь:** каждый алгоритм сетки — отдельный класс-стратегия (`DoubleEliminationStrategy`), стадии описаны конечными автоматами (aasm) на трёх уровнях, а оркестрация вынесена в ежедневную задачу clockwork.

## Бенефиты
- Новый тип турнира = новый класс-стратегия, без правки `if`-каши.
- Переходы стадий явные и защищённые (aasm), а не разрозненные флаги.
- Голоса с кешем через `acts_as_votable`.
- Оркестрация процесса отделена в cron-задачу.
