# 32 — `acts_as_list` (позиционирование)

## Простыми словами
Сам следит за порядком записей в списке через колонку `position` — добавил, удалил или перетащил элемент, а позиции остальных пересчитались автоматически. И всё это в рамках группы (например, у каждого пользователя свой порядок избранного). Это как нумерованная очередь: кто-то вышел из середины — и всех за ним подвинули, не пересчитывая номера вручную.

## Что
Авто-управление порядком записей через колонку `position`, в рамках скоупа.

## Реальный код
```ruby
class Favourite < ApplicationRecord
  acts_as_list scope: %i[user_id linked_type kind]   # порядок внутри группы
  scope :ordered, -> { order :position }
end
```

## Как работает
- `scope:` — внутри какой группы считается позиция (тут: у юзера, по типу и виду избранного).
- Автоматом проставляет/сдвигает `position` при создании/удалении/перемещении.
- Методы: `move_higher`, `move_to_top`, `insert_at(n)` и т.п.

## Зачем / где полезно
Сортируемые списки: избранное, пункты меню, шаги, плейлисты, drag-and-drop порядок. Не надо руками пересчитывать позиции при вставке в середину. `scope` изолирует нумерацию по группам.

## 🆚 Классический Rails
**Обычно:** колонку `position` ведут руками — при вставке в середину или удалении приходится самому пересчитывать и обновлять позиции остальных записей, легко наплодить багов и дыр в нумерации.
**Здесь:** `acts_as_list` берёт это на себя: автоматически проставляет и сдвигает `position`, причём в рамках заданного `scope` (группы).

## Бенефиты
- Не надо вручную пересчитывать позиции при вставке/удалении/перемещении.
- Готовые методы перестановки: `move_higher`, `move_to_top`, `insert_at(n)`.
- `scope` изолирует нумерацию по группам (у каждого юзера свой порядок).
- Идеально под drag-and-drop сортировку на фронте.
