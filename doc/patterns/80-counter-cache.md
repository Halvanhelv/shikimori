# 80 — Денормализованные счётчики (counter cache)

## Простыми словами
Чтобы не считать `COUNT(*)` комментариев на каждый показ темы, хранят готовое число в колонке `comments_count` и обновляют при добавлении/удалении. Показ — мгновенный.

## Реальный код
```ruby
# тема хранит готовый счётчик
topic.comments_count       # 0 запросов — просто читаем колонку

# acts_as_votable с кешем голосов (см. 33):
acts_as_votable cacheable_strategy: :update_columns
# cached_votes_up / cached_votes_down хранятся в колонках

# обновление счётчика при создании коммента:
def touch_topic(comment) = comment.commentable.touch
```

## Как работает
- В таблице есть колонка-счётчик (`comments_count`).
- При создании/удалении связанной записи счётчик инкрементится/декрементится (Rails `counter_cache: true` или вручную).
- Чтение счётчика = чтение колонки, **без** `COUNT(*)`.

## 🆚 Классический Rails
**Обычно (наивно):** `topic.comments.count` на каждый рендер → `SELECT COUNT(*)` каждый раз. На списке из 50 тем — 50 COUNT-запросов (N+1 по счётчикам).
**Здесь:** число уже в колонке → ноль запросов на подсчёт, список тем рендерится без COUNT'ов.

## Бенефиты
- Нет дорогих `COUNT(*)` на горячих страницах.
- Списки рендерятся без N+1 по счётчикам.
- Можно сортировать/фильтровать по счётчику с индексом.

## Когда не надо
Счётчик меняется очень часто и точность не критична на каждый момент → можно считать периодически. Денормализация требует дисциплины: счётчик надо поддерживать в синхроне (иначе «врёт»). Для редких подсчётов — обычный `.count`.
