# 93 — Server-rendered + Turbolinks (модель фронта)

## Простыми словами
Shikimori — не SPA на React. Страницы рендерит сервер (HTML на Slim-шаблонах), а Turbolinks делает переходы между ними быстрыми: подменяет содержимое страницы без полной перезагрузки, сохраняя ощущение «приложения».

## Реальный код
```slim
/ app/views/animes/show.html.slim — сервер рендерит готовый HTML
.b-anime
  = render 'poster', anime: anime
  a.add-favourite data-remote='true' href=favourite_path(anime)   / AJAX-действие
```
```ruby
# контроллер определяет turbolinks-запрос (см. 45):
def turbolinks_request? = request.headers['X-XHR-Referer'].present?
```

## Как работает
- Сервер отдаёт полный HTML страницы (Slim-шаблоны).
- Turbolinks перехватывает клики по ссылкам → грузит новую страницу AJAX'ом → подменяет `<body>` без полной перезагрузки (JS/CSS не перезагружаются).
- Точечная интерактивность — через `data-remote` (AJAX) и островки Vue ([94](94-vue-islands.md)).

## 🆚 SPA-подход (React/Vue целиком)
**SPA:** весь UI на клиенте, сервер отдаёт только JSON. Плюс — богатая интерактивность; минус — сложность, дублирование логики, SEO/первый рендер, большой JS-бандл.
**Здесь (SSR + Turbolinks):** сервер — единственный источник HTML, JS — «приправа». Проще, SEO «из коробки», меньше дублирования. Минус — меньше клиентской интерактивности.

## Бенефиты
- Простота: одна кодовая база рендерит HTML, нет дубля логики на клиенте.
- SEO и быстрый первый рендер бесплатно.
- Быстрые переходы без полной перезагрузки (Turbolinks).

## Когда не надо
Очень интерактивный UI (редакторы, дашборды реального времени) → там SPA/Hotwire-Turbo оправданы. Для контентного сайта (каталог, форум) SSR+Turbolinks — отличный баланс. Современный наследник — Turbo (Hotwire).
