# 74 — Translation concern (короткий доступ к i18n)

## Простыми словами
Чтобы не писать длинное `I18n.t('очень.длинный.ключ')` везде, заводят общий модуль с короткими хелперами перевода, который подмешивают в контроллеры, вьюхи и сервисы.

## Реальный код
```ruby
class ApplicationController < ActionController::Base
  include Translation       # даёт i18n_t, i18n_i и т.п.
  helper_method :i18n_i, :i18n_io, :i18n_v
end

# в mailer'е:
class ShikiMailer < ActionMailer::Base
  include Translation
  subject = i18n_t('reset_password_instructions.subject', locale: user.locale.to_sym)
end
```

## Как работает
- `Translation` — модуль-примесь с методами-обёртками над `I18n.t` (с авто-неймспейсом ключей, подстановкой локали).
- Подмешивается туда, где нужны переводы: контроллеры, мейлеры, сервисы, вью-объекты.
- Короткие имена (`i18n_t`, `i18n_i`) вместо длинных вызовов.

## 🆚 Классический Rails
**Обычно:** в контроллерах/вью есть `t(...)`, но в сервисах/PORO его нет — пишут полный `I18n.t('...')`, дублируют неймспейс ключа.
**Здесь:** один concern даёт единый короткий API перевода **везде**, включая обычные объекты.

## Бенефиты
- Единый способ переводить во всех слоях (не только во вьюхах).
- Меньше дублирования длинных ключей/неймспейсов.
- Легко добавить логику (fallback, локаль получателя — см. [51](51-mailer-guards-i18n.md)).

## Когда не надо
В чистых вьюхах/контроллерах хватает встроенного `t`. Concern окупается, когда переводы нужны в сервисах/мейлерах/PORO.
