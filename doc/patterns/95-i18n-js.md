# 95 — i18n-js (переводы Rails → JavaScript)

## Простыми словами
Переводы (`config/locales`) нужны не только на сервере, но и в JS (надписи в Vue-островах, сообщения JS-виджетов). `i18n-js` экспортирует те же словари в JavaScript, чтобы не дублировать строки.

## Реальный код
```ruby
# config/locales/*.yml — единый источник переводов
ru:
  anime:
    add_to_list: "Добавить в список"
```
```js
// в JS те же ключи:
I18n.t('anime.add_to_list')   // "Добавить в список"
```

## Как работает
- `i18n-js` берёт серверные YAML-словари и генерит из них JS-версию.
- На клиенте доступен тот же `I18n.t('ключ')` с теми же ключами и локалями.
- Локаль текущего юзера прокидывается на фронт.

## 🆚 Классический подход
**Обычно (плохо):** строки для JS пишут отдельно (хардкод в .js или отдельный json) → перевод есть на сервере, но в JS захардкожен по-английски; добавил язык — забыл про JS.
**Здесь:** один источник переводов для бэка и фронта. Добавил ключ в YAML — доступен и там, и там.

## Бенефиты
- Нет дублирования строк сервер/клиент.
- Новый язык/ключ автоматически доступен в JS.
- Единая система плюрализации/интерполяции.

## Когда не надо
Если на фронте почти нет текста (чистый SSR) — экспорт лишний. Окупается при заметной клиентской части (Vue-острова, JS-виджеты с надписями).
