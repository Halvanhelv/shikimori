# 71 — `.delay` (отложить любой вызов в фон)

## Простыми словами
Иногда нужно выполнить метод не сейчас, а в фоне, но лень заводить отдельный класс-воркер. `.delay` позволяет «отложить» любой вызов модели/метода в очередь Sidekiq.

## Реальный код
```ruby
# вместо немедленного:
SomeModel.heavy_recalculation(id)

# отложенно в фон (sidekiq-delay_extensions):
SomeModel.delay.heavy_recalculation(id)
SomeModel.delay_for(5.minutes).heavy_recalculation(id)   # с задержкой
```

## Как работает
- Гем `sidekiq-delay_extensions` добавляет `.delay` к классам/методам.
- Вызов сериализуется (класс, метод, аргументы) и кладётся в очередь.
- Sidekiq-воркер потом десериализует и выполняет.

## 🆚 Классический Rails
**Обычно:** под каждую фоновую задачу заводят отдельный класс-Job (`SomeJob < ApplicationJob`) — много файлов ради разовых вещей.
**Здесь:** `.delay.method` — фоновое выполнение без отдельного класса. Быстро для разовых/простых задач.

## Бенефиты
- Не плодить классы-воркеры под каждую мелочь.
- Отложить можно любой существующий метод.

## Когда не надо
Для важных/частых задач лучше **явный** Job-класс: его видно, легко тестировать, ставить лимиты/локи ([19](19-sidekiq-unique-jobs.md), [20](20-sidekiq-limit-fetch.md)). `.delay` сериализует аргументы — большие объекты слать нельзя (только id). В новых Rails — `perform_later`.

⚠️ В проекте включён с фиксом безопасной YAML-десериализации (Ruby 3) — `.delay` исторически был источником уязвимостей при небезопасной сериализации.
