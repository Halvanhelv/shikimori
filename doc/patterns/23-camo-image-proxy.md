# 23 — Camo: прокси картинок

## Простыми словами
Чужие картинки показываются не напрямую, а через свой сервер-посредник, который сам их скачивает и отдаёт пользователю. Это как заказ через службу доставки: продавец не узнаёт твой адрес, потому что общается только с курьером.

## Что
Все внешние картинки (с booru-сайтов) идут через HMAC-подписанный прокси (`camo-server`, отдельный сервис). Решает три проблемы разом.

## Реальный код
```ruby
# подпись url ключом — нельзя подделать/перебрать
OpenSSL::HMAC.hexdigest('sha1', CAMO_KEY, url)

# балансировка статики по CDN-поддоменам через id % N
image_index = entry.id % SELECTED_STATIC_SUBDOMAINS.size
production_url(image_url_path, image_index)   # → nyaa.shikimori.one / dere.shikimori.one

# доверенные домены проходят мимо прокси:
NON_CAMO_DOMAINS = /\A(?: shikimori\.(org|one|dev) | imgur.com | cloudinary.com | ... )\Z/mix
```

## Как работает
- Картинка с чужого домена → подменяется на `camo`-URL с HMAC-подписью.
- Камо качает оригинал на сервере и отдаёт юзеру → IP юзера не уходит на чужой сервер.
- `id % N` равномерно раскидывает статику по поддоменам (больше параллельных коннектов в браузере).

## Зачем / где полезно
- **Privacy**: не утекает IP/реферер юзера на внешние хосты.
- **Mixed-content**: https-сайт не ломается на http-картинке.
- **CDN-балансировка**: модуло-хеширование по id.
- HMAC не даёт подделать URL или перебирать чужие.

Учит: проксирование внешнего контента как слой безопасности + распределение нагрузки.

## 🆚 Классический Rails
**Обычно:** внешние URL картинок вставляют во вьюху как есть (`image_tag remote_url`) — браузер юзера сам ходит на чужой сервер, светя IP и реферер.
**Здесь:** все внешние картинки проходят через HMAC-подписанный camo-прокси, а статика ещё и раскидывается по CDN-поддоменам через `id % N`.

## Бенефиты
- Приватность: IP/реферер юзера не уходят на сторонние хосты.
- Нет mixed-content: http-картинка не ломает https-страницу.
- Балансировка статики по поддоменам → больше параллельных коннектов.
- HMAC-подпись не даёт подделать URL или перебирать чужие.
