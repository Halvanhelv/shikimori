# 38 — Русский i18n + словоформы

## Простыми словами
Перевод на русский — это не просто «словарь английский → русский». Язык живой: «3 января», но «января» в одном месте и «январь» в другом, «Аня» в родительном падеже звучит иначе. Это как живой переводчик, который знает не только слова, но и грамматику: правильно склоняет, выбирает падеж и даже учитывает, хочет ли сам юзер видеть русские названия.

## Что
Полноценная поддержка русского сверх перевода строк: падежи месяцев, склонения, транслит, двуязычные названия по предпочтению юзера.

## Реальный код
```ruby
# Gemfile — форк гема для русской морфологии
gem 'russian', github: 'morr/russian', branch: 'master'

# падежи месяцев (родительный для дат)
common_abbr_month_names: [nil, 'янв.', 'февр.', 'марта', 'апр.', 'мая', ...]

# выбор названия по локали + предпочтению юзера
def entry_name(entry, user)
  is_russian = I18n.russian? && (!user || user.preferences.russian_names?)
  is_russian && entry.russian.present? ? entry.russian : entry.name
end
```

## Как работает
- Гем `russian` даёт морфологию: падежи, склонения, правильные формы месяцев.
- `i18n-inflector` (форк) — сложная плюрализация для нескольких языков.
- `translit` — кириллица → латиница для имён файлов.
- Двуязычные сущности: показываем `russian` или `name` по локали и настройке юзера.

## Зачем / где полезно
Продакшн-поддержка не-английских языков с богатой морфологией. Учит: i18n ≠ только словарь строк — это падежи, склонения, словоформы, пользовательские предпочтения отображения.

## 🆚 Классический Rails
**Обычно:** встроенный `I18n` + `config/locales/*.yml` — это плоский словарь «ключ → строка» с простой плюрализацией (`one`/`few`/`many`). Падежи, склонения имён, транслит — из коробки нет.
**Здесь:** поверх стандартного `I18n` подключены гем `russian` (морфология: падежи месяцев, склонения), форк `i18n-inflector` (сложная плюрализация) и `translit`. Плюс двуязычные сущности: название показывается на русском или языке оригинала по локали и настройке юзера.

## Бенефиты
- Грамматически правильный русский (даты, склонения), а не «3 январь».
- Пользователь сам выбирает русские/оригинальные названия.
- Транслит для ЧПУ-урлов и имён файлов.
- Логика выбора названия в одном месте (`entry_name`), а не размазана по вьюхам.
